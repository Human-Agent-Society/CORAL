System Overview

CORAL is an orchestration system that turns autonomous coding agents into persistent optimizers. Rather than single-shot code generation, CORAL sets up a continuous cycle: agents independently explore solutions, evaluate them against a scoring function, and improve — indefinitely and without human intervention.

Architecture

CORAL's architecture has four layers:

  1. Task & Evaluation. A CORAL run begins with two definitions: a Task (what to solve — a codebase, a configuration, and a description) and an Evaluation (how to score — a grader that produces a score and feedback for each attempt). Together, these define the optimization problem.
  2. Management Infra. The management infra manages the full agent lifecycle — spawning agents, monitoring their health, restarting them when they exit, and delivering evaluation feedback. A heartbeat system periodically interrupts agents for forced reflection, consolidating shared information so the whole team benefits from individual discoveries.
  3. Agent Pool. Multiple agents run in parallel, each in its own isolated git worktree. Every agent follows the same workflow loop (research — plan — edit — evaluate — read feedback — repeat) but is free to pursue a different strategy. This parallel exploration searches a much wider solution space than any single agent could.
  4. Shared Knowledge. Agents are isolated in their worktrees but connected through a shared knowledge layer consisting of three components:
    • Attempts — a scored log of every experiment, forming a leaderboard.
    • Notes — cross-agent learning that helps others avoid dead ends and build on discoveries.
    • Skills — reusable strategies that agents package up for others to apply.

The power of CORAL is in the cycle. Each iteration produces a scored commit, visible to all agents. Good strategies surface through the leaderboard. Insights propagate through shared notes. Over time, the system converges on strong solutions through a process closer to collaborative research than one-shot generation.

Heartbeat System

As agents continue to run, their workspace slowly becomes messier — like an uncleaned office table after many days of work. The heartbeat system provides periodic reminders for agents to "clean" themselves up: reflecting on their experience, writing notes, and consolidating knowledge into shared skills. This serves as a librarian, providing a centralized place for agents to exchange ideas and know-how, ensuring they can work perpetually without accumulating unnecessary context noise.

Experiments

We evaluated CORAL on four tasks spanning complex mathematics, ML engineering, and GPU kernel engineering. Our primary baseline is OpenEvolve, the leading open-source implementation of Google DeepMind's AlphaEvolve — an LLM-driven evolutionary system that optimizes code through population-based search.

While both CORAL and AlphaEvolve/OpenEvolve have similar outer loops, the mechanisms underneath are fundamentally different. OpenEvolve evolves marked code blocks through LLM-guided mutations, maintaining candidates via MAP-Elites. In CORAL, each agent is fully autonomous: it reads the entire codebase, possesses its own workspace, reasons over its own history of failed attempts, and with multiple agents, learns from and contributes to the discoveries of others.

Both systems were given identical seed programs and evaluation functions. No human intervention occurred after launch. All experiments used Claude Opus 4.6 as the backbone model.

Kernel Engineering: Anthropic's Performance Take-Home

This task involves optimizing a tree traversal algorithm on a simulated VLIW SIMD machine, starting from code that performs at 18,532 cycles. In the 2-hour interview setting, the approximate best human score was 1,790 cycles. The previous best known performance was 1,363 cycles, achieved by Claude Opus 4.5 with an improved test-time harness.

CORAL broke the previous best known score, with a single agent reaching 1,350 cycles in just 4.6 hours. Interestingly, OpenEvolve with the same backbone model was unable to beat the 1,363 threshold, struggling with a task that demands coordinated, multi-step rewrites rather than local mutations.

When scaling to four parallel agents, the multi-agent configuration drove the score down to 1,103 cycles — a 24% improvement over the single-agent result, with no additional time cost.

Mathematics: Erdős' Minimum Overlap Problem

This is a combinatorial number theory problem where the aim is to find a step function that minimizes the maximum overlap when the integers {1, 2, …, 2n} are split into two sets of size n and their sumsets are considered. Due to its difficulty as an open problem, it is commonly used as a benchmark for agentic systems. The score used is C∞, where lower is better.

C∞ = 0.38078
New SOTA for Erdős' Minimum Overlap Problem — surpassing AlphaEvolve's 0.38092
Figure 3. CORAL reaches 99% of AlphaEvolve's score in just 34 minutes (vs. 5.2 hours for OpenEvolve). With web-search enabled, near-optimal solutions emerge in as little as 6 minutes.

CORAL (without web-search) finds a solution with C∞ = 0.38092, matching the best reported AI solution by AlphaEvolve and outperforming OpenEvolve's 0.38111. More importantly, CORAL requires far fewer evaluations and converges much faster.

Allowing agents to access web-search further improves both performance and convergence speed, achieving C∞ = 0.38078. With external knowledge retrieval, CORAL discovers relevant mathematical techniques from the literature and incorporates them during the search process.

ML Engineering: Kaggle Spaceship Titanic

To test the more common scenario — a practitioner throwing an agent at a well-understood ML problem — we used Spaceship Titanic from Kaggle. The task is binary classification on tabular passenger data. While categorized as a beginner competition, achieving top leaderboard scores requires systematic feature engineering, model selection, and ensemble techniques.

We evaluate on an MLEBench custom split (90/10 from the original training data) with classification accuracy as the metric. The seed program is a Random Forest with basic feature engineering.

Multi-Agent Collaboration

The CORAL architecture naturally supports multiple agents that collaborate with each other. Unlike traditional multi-agent harnesses that hardcode workflows, CORAL's autonomous agents run together with respectful privacy. Each agent researches its own ideas, but can peek (not modify) others' workspaces to learn what they're doing.

An emergent behavior we observed: when an agent makes a breakthrough by submitting a new SOTA, other agents soon follow up to apply the technique to their own methodologies for even better solutions. A virtuous cycle is created where a group of agents evolves significantly faster than a single agent alone.

We observe that as evolution progresses, agent collaboration becomes increasingly flexible. During the early stage, notes are simple summaries. The interesting phenomenon appears later: when all agents have converged, they naturally work more closely, collectively forming a consensus via shared notes.

[Figure: Multi-agent collaboration timeline — early vs. late stage notes]
Figure 5. Evolution of inter-agent collaboration. Early notes are independent summaries. Later notes show agents analyzing each other's strategies, highlighting discoveries marked "CRITICAL", and forming consensus.

At stuck regions, an agent actually tried to analyze the difference between the other two agents and highlight the discovery to others (marked with "CRITICAL"). Such behavior demonstrates the potential for agents to naturally evolve into different roles that handle heterogeneous tasks while contributing to a common goal.

Towards a Human-Agent Society

To gain deeper insight into CORAL's continuous evolution, we built a comprehensive web interface that traces every agent action, shared workspace, and evaluation metric in real-time. As we observe this progression, the impulse to intervene often arises — whether to encourage out-of-the-box thinking when agents get stuck or to offer guidance based on our own domain knowledge.

Just as the agents frequently surprise us by pushing the boundaries of human capability, we find ourselves wanting to reciprocate with our own feedback. Ultimately, we are moving toward a collective intelligence where humans and agents collaborate and co-evolve in distinct but complementary roles to achieve shared goals.
